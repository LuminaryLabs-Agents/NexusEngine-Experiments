# Forward Abstraction Workspace

This workspace moves one validated output forward at a time. It does not replace historical production or saturation ledgers.

```txt
validated playable
  -> application-domain kit
  -> atomic responsibility map
  -> kit/subdomain taxonomy
  -> engine-domain consolidation
  -> one-layer semantic promotion
  -> consumer migration + parity
  -> proven duplicate purge
  -> derived stress experiment
```

## Files

- `stage-ledger.md` — the single active artifact, current/target layer, proof, and next stage.
- `contracts.md` — mandatory extraction, export, migration, and purge gates.
- `records/` — machine-readable per-artifact ownership and stage records.
- `.agent/tasks.jsonl` — append-only user-request status events.

Only one row may be active in the stage ledger. A cycle may prepare later gates, but it may advance only the declared current stage.
