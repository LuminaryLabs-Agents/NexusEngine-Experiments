# SAT-005 — Counterweight Cathedral

Status: validated / kit-reviewed / preserved
Disposable/preserved root: `experiments/counterweight-cathedral/`
Selected: 2026-07-19

## Immutable Prompt

Tune a suspended cathedral mobile by moving three weighted stones along a rail and settling each into the bowl whose authored threshold it exactly satisfies.

## Semantic Boundary

Primary stress edge: deterministic lightweight body impulses establish stone positions, which become explicit weighted-trigger sources whose aggregate facts activate three independent bowls.

The route owns stone/bowl fiction, selection, socket index presentation, settle command composition, exact-weight success/failure, input, and rendering. It must not own parallel body position, velocity, plate weight, source membership, or activation state.

## Player Loop

```txt
select stone -> shift left/right along rail -> inspect target bowl -> settle weight
-> avoid overloading any bowl -> activate all three exact thresholds -> cathedral chord or fracture -> restart
```

## Quality Bar

- Goal, three stones, three bowls, and three hero actions read within five seconds.
- Selected stone, physical position, bowl requirement/current weight, activation, overload, and ending are visible without diagnostics.
- Settling the wrong heavy stone creates a clear loss; the exact 2/3/1 arrangement creates a clear success.
- Suspended sculpture presentation and spatial placement are distinct from maps, timing meters, ecology, councils, and rescue.
- Advanced evidence stays closed by default and restart is deterministic.

## Lessons

- Body impulses and trigger sources remain cleanly separated when the host submits an explicit settle observation.
- Exactness is a route rule over canonical aggregate weight, not a second trigger implementation.
- Five unrelated player loops now failed to prove a Core/atlas gap after consolidation.
