# ProtoKit Promotion Gate — 2026-06-24 03:30

## Lens

Promotion is not gap filling. A reusable behavior is promotion-worthy only when it becomes a domain communication boundary through resources, events, methods, snapshots, and descriptors while staying headless where possible and keeping renderer/browser ownership out of reusable logic.

## `.agent` memory read before change

- ProtoKits memory says the long-form goal is reusable DSK-based ProtoKits while shrinking local experiment JavaScript.
- Experiments memory says `signal-bastion` should remain the only executable route-domain lane for now, and the next safe strategic-pressure work is one guarded browser-host convenience-facade shrink at a time.
- Core `.agent/intent.md` is still not present on `main`, so Core durable memory remains a known gap.

## Change made

Shrank one Signal Bastion browser-host convenience seam without moving reusable kit implementation into Experiments:

- `games/signal-bastion/src/boot.js` now derives `GameHost.getWavePreview()` from `engine.n.genericDefense.sessionFacade.getSnapshot()` through `getSignalBastionWavePreview(engine)`.
- `GameHost.getWavePreview()` no longer calls `engine.defenseWaves.previewNextWave()`.
- `tests/signal-bastion-host-facade-guard-smoke.mjs` now forbids returning to `engine.defenseWaves.previewNextWave()` and treats wave preview as a namespaced DSK-session snapshot read.
- `experiments/signal-bastion-route-domain-replay.json` records this as local JS / facade shrink while keeping foundation, build blueprint/sell, and scale as explicit remaining convenience seams.

## Boundary result

DSK boundaries became clearer for the strategic-pressure lane:

- Wave preview is now derived from the generic-defense session snapshot rather than a broad wave facade.
- Wave start, upgrade, restart, selection, snapshot, placement confirmation, render descriptor fallback, and wave preview are now aligned around `engine.n.genericDefense.sessionFacade` / `engine.n.genericDefense.renderDescriptors`.
- The route still owns browser input, Canvas/HUD projection, and presentation hooks only.

## Local JavaScript reduction

Small but real: the browser host lost one broad convenience-facade call (`engine.defenseWaves.previewNextWave`) and now uses the namespaced DSK snapshot already required by the executable strategic-pressure replay.

## Higher-level domains

The only executable route-domain lane remains strategic pressure loop: defense map/slots/vital + economy + build placement + wave/agent director + combat + render descriptors. Delivery/extraction, survey pressure, field-engineer composition, aerial traversal, survival ecology, and action-defense extraction remain contract/planned until a real reusable ProtoKit boundary is consumed by an experiment route.
