# Nexus Engine Forward-Abstraction Goal

Status: active

## Mission

Move the latest validated stress-experiment output forward through explicit abstraction stages. Each bounded cycle consumes the prior artifact, advances one stage, proves parity, and leaves the next stage unambiguous. New experiments are derived only after the promoted domains are ready to be stressed again.

## Preserved Baseline

The completed capability-saturation series remains authoritative under `.agent/saturation-series/`: SAT-001 through SAT-005 reached `5 / 5` with zero newly accepted atomic capabilities. That result is historical proof, not permission to create SAT-006 or discard its playables.

## Output-Forward Loop

```txt
validated playable
  -> local full-game application-domain kit
  -> atomic responsibility map
  -> canonical kit/subdomain ownership
  -> engine-domain consolidation
  -> exactly one semantic abstraction promotion
  -> consumer migration + parity
  -> duplicate retirement/purge when proven safe
  -> next distinct stress experiment derived from the promoted boundary
```

Reusable semantic promotion follows `implementation -> behavior -> kit -> subdomain -> domain -> cross-domain capability`, exactly one step per cycle. Browser, renderer, camera, audio, visual effects, DOM, network, storage, scenes, and authored tuning remain explicit local/application domains; they are not pushed into reusable simulation domains merely to raise a layer.

## Cycle Gates

- Use Core first, existing engine domains second, and accept a new atomic capability only with Core-gap evidence, one semantic responsibility, canonical ownership, and multiple plausible consumers.
- Track each candidate as `Core-reused`, `domain-reused`, `newly accepted`, `merged duplicate`, `specialization`, `adapter-only`, `local-only`, or `rejected`.
- Advance one artifact and one stage per bounded cycle; do not perform whole-repository consolidation.
- Export the current artifact with its ownership, dependencies, consumers, proof, unresolved gaps, and exact next stage.
- Do not purge a source until successor coverage, consumer migration, reference cleanup, deterministic parity, playable parity, and performance parity all pass.
- Preserve validated playables as proof unless a later migration supplies an equivalent validated successor.
- Derive the next experiment from an unresolved boundary in the consolidated domain atlas; unrelated quota-filling does not count.
