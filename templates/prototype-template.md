## Prototype Goal

What one question should this prototype answer?

## Prototype Rule

Build only what is needed to test the core loop.

## Must-Have Features

- Player can start a run.
- Player can perform the core action.
- Player can score or progress.
- The game can escalate.
- The run can end.
- The player can restart.

## Nice-To-Have Features

- Extra enemy types.
- Extra levels.
- Juice / polish.
- Better menus.
- Cabinet attract mode.

## Test Loop

```txt
Launch prototype.
Play for 60 seconds.
Record what felt good.
Record what was confusing.
Adjust scope.
Repeat.
```

## Success Criteria

The prototype succeeds if:

- The core loop is understandable within 5 seconds.
- The controls feel immediate.
- The player wants one more run.
- The scoring or progression is clear.

## Failure Criteria

The prototype fails if:

- The idea is too confusing.
- The controls require explanation.
- The loop is not fun after one minute.
- The cabinet fit is weak.
